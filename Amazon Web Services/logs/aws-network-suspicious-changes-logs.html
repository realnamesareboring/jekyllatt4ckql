Detection Logic Explanation

This detection identifies network security configuration changes made from untrusted IP addresses or known malicious IPs. It focuses on modifications to security groups, network ACLs, routes, and other network components that could indicate unauthorized access attempts or potential compromise.

Key indicators in the logs:

  • The EventName field shows what type of network configuration change occurred
  • The DetailSummary provides a comprehensive description of the change, including security implications
  • Modifications that open access to 0.0.0.0/0 (all IPs) are highlighted as particularly concerning
  • Changes to port ranges, especially for sensitive ports like 22 (SSH) or 3389 (RDP)
  • The SourceIpAddress indicates the origin of the change request, which may be untrusted
  • All events have a Medium severity based on the query filtering
  • Network visibility changes like DeleteFlowLogs could indicate attempts to hide malicious activity
  • VPC endpoint modifications may indicate attempts to create unauthorized data paths

Recommended Response

  • Immediately review all security group modifications, especially those allowing:
    • Inbound traffic from 0.0.0.0/0 (anywhere)
    • Access to management ports (SSH 22, RDP 3389)
    • Overly permissive port ranges
  • Verify if the network changes were authorized by:
    • Consulting with the change management system
    • Checking with the users who made the changes
    • Reviewing any automation scripts that might have triggered these changes
  • For changes from untrusted or known malicious IPs, treat as a potential compromise:
    • Immediately revert unauthorized changes
    • Enable flow logs if they were disabled
    • Scan affected instances for signs of compromise
  • Implement preventative controls:
    • Service Control Policies (SCPs) to restrict critical network changes
    • Require MFA for sensitive network operations
    • Implement VPC endpoint policies with least privilege
    • Create automated alerts for suspicious network changes
  • Check for additional suspicious activity by the same users/roles
  • Ensure flow logs are enabled for all VPCs to maintain visibility
